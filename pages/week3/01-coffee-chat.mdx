### `==` vs `===`의 차이점

많은 사람들이 double equals과 triple equals의 차이점에 대하여,

- ~Double equals은 값을 비교합니다.~
- ~Triple equals은 값과 자료형을 모두 비교합니다.~

이렇게 알고 계시는 분들이 많습니다. 하지만 이것은 **완전히 잘못된 정보**입니다.

<br />
<Alert
  showIcon
  message="== vs ==="
  description="두 가지의 차이점은 Type coercion (암묵적 형변환)에 있습니다."
  type="info"
/>

정확히 말하자면,

- Double equals는 (필요할 경우) **암묵적으로 형변환 과정**을 거친 후, 값을 비교합니다.
- Triple equals는 형변환 과정 없이, 값을 비교합니다.

<br />
<Alert
  showIcon
  message="사용 권장 컨벤션"
  description="정말 특별한 이유가 있지 않다면, triple equals의 사용이 항상 강력하게 권장됩니다."
  type="warning"
/>

기초적인 상식이지만, 생각보다 매우 많은 경력자, 쥬니어 등의 개발자들이 놓치는 부분이니 확실하게 알고 계시기 바랍니다.

### 함수의 반환값

함수의 반환값에 대해 정확히 이해하고, 우리의 코드에서 반환값이 어떻게 전달되고 연결되는지를 파악하는 것은 매우 중요한 부분입니다. 하지만 제가 지금까지 봐온 대부분의 초보자들이 반환값을 이해했다고 말하면서도 실제 코드에서는 정확하게 판별하지 못하는 모습을 보이곤 합니다. 확실하게 이해하는 시간을 갖도록 해봅시다.

한 마디로 요약하자면, **함수 실행 구문은 해당 함수의 반환값으로 치환**됩니다.

기본적인 예시부터 살펴보겠습니다.

<br />
<br />

##### 예시 1-1.

```js showLineNumbers /add(1, 4)/ copy
function add(a, b) {
  return a + b;
}

const result = add(1, 4);
```

위의 예시 코드에서 함수 실행 구문 `add(1, 4){:js}`는 해당 함수 실행 구문의 반환값으로 치환됩니다. 즉, `add` 함수가 실행된 이후에 5로 치환되어 아래와 동일한 코드라고 볼 수 있습니다.

##### 예시 1-2.

```js showLineNumbers /5/ copy
function add(a, b) {
  return a + b;
}

const result = 5;
```

<br />
<br />

##### 예시 2-1.

```js showLineNumbers /: getName()/ copy
function getName() {
  return "ken huh";
}

const person = {
  name: getName(),
};
```

위의 예시 코드에서 `person` 객체가 생성되는 과정에서 함수 실행 구문 `getName(){:js}`이 실행됩니다. 그리고 이 실행문은 해당 함수 실행 구문의 반환값으로 치환됩니다. 즉, `getName` 함수가 실행된 이후에 `"ken huh"`로 치환되어 아래와 동일한 코드라고 볼 수 있습니다.

##### 예시 2-2.

```js showLineNumbers /: "ken huh"/ copy
function getName() {
  return "ken huh";
}

const person = {
  name: "ken huh",
};
```

<br />
<br />

너무 쉽다고요? 조금 더 복잡한 예시 코드를 한번 살펴볼까요?

##### 예시 2.

```js showLineNumbers /curry()/ copy
function curry() {
  return function curried() {};
}

const something = curry();
```

`curry()` 실행 구문은 해당 함수의 반환값으로 치환됩니다. 근데 눈여겨 봐야 할 부분이 있습니다. 바로, 해당 함수의 반환값이 참조값이라는 것입니다. 이런 경우, 포스트잇에 적힌 해당 자료의 위치값이 반환된다고 생각하셔야 합니다.

현재 `curry` 함수는 실행되면, `curried` 함수를 선언(생성)하고 해당 함수의 위치값을 반환합니다. 그 위치값을 88번이라고 가정했을때, `something` 변수에는 88번이라는 위치값이 대입되는 것입니다.

<br />
<br />

##### 예시 3.

```js showLineNumbers /curry(add)/ /something(2, 2)/ /fn(a, b)/ copy
function add(a, b) {
  return a + b;
}

function curry(fn) {
  return function curried(a, b) {
    return fn(a, b);
  };
}

const something = curry(add);
const result = something(2, 2);
```

1. 11번째 줄의 `curry(add)` 함수 실행 구문은 `curried` 함수의 위치값으로 치환됩니다. 즉, `curried` 함수가 됩니다. (`fn`은 `add` 함수입니다.)
2. 11번째 줄의 `curry(add)` 함수 실행 구문이 `curried` 함수의 위치값이기 때문에, `something` 변수에 해당 위치값이 대입됩니다.
3. 그렇기 때문에, 12번째 줄의 `something(2, 2)` 함수 실행 구문은 `curried(2, 2)`와 같다고 볼 수 있습니다.
4. `something(2, 2)` 또한 함수 실행 구문이기 때문에, 해당 함수(`curried(2, 2)`)의 반환값으로 치환됩니다.
5. `curried` 함수가 실행되면, 내부 로직인 `fn(a, b)`가 실행됩니다. 현재 `fn`은 `add` 함수이고, `a`와 `b`는 모두 2입니다. 그러므로 `curried` 함수 내부에서는 `add(2, 2)`가 실행되어 반환됩니다.
6. `add(2, 2)` 역시 함수 실행 구문이기 때문에 해당 함수 실행문의 반환값인 4로 치환됩니다. 그러므로 `curried` 함수에서는 4가 반환됩니다.
7. 3번 단계에서 확인한 내용에 따르면, `something` 함수와 `curried` 함수가 같은 상황이기 때문에 12번째 줄의 `something(2, 2)` 구문은 4로 치환됩니다.
8. 결과적으로 `result`는 4가 됩니다.

<br />
<br />

##### 예시 4.

```js showLineNumbers /curriedAdd(1)/ copy
function add(a, b, c) {
  return a + b + c;
}

function curry(fn) {
  return function (a) {
    return function (b) {
      return function (c) {
        return fn(a, b, c);
      };
    };
  };
}

const curriedAdd = curry(add);
const result = curriedAdd(1)(2)(3);
```

위의 예시 코드에서도 복잡할 것은 크게 없습니다. `curriedAdd(1)`, `curriedAdd(1)(2)` 이 구문들이 모두 함수를 반환하고 있고 그렇기에 다시 함수 실행문으로 연결된 것 뿐입니다.
