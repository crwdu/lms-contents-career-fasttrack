# 📗 문제에 대처하는 방법

<br/>
## 1. 당황하지 말고 차분하게 임하기 🥸

가장 중요한 부분입니다. 정신이 산만하고, 흥분하거나 평정심을 잃는다면 이미 문제해결을 위한 과정의 절반은 실패라고 볼 수 있습니다. 차분함과 평정심을 유지하고, 문제를 냉정하게 바라볼 수 있도록 마음을 추스리세요.

(T 같나요..?)

<br/>
## 2. 개발자 도구 콘솔 확인하기 🖥️

크롬 브라우저의 개발자 도구를 열어서 `console` 탭에 혹시 **오류 메시지가 출력되어 있는지 확인**해봐야 합니다. 만약 오류 메시지가 있다면, 오류 메시지 구문의 의미를 100% 정확히 이해해야 합니다.

<Callout emoji="📌">
  개발자도구를 어떻게 열 수 있는지 모르신다면, "크롬 개발자도구 열기"라고 구글에 검색해보세요. 🙂
</Callout>

<br/>
## 3. 샘플 코드와 면밀히 비교해보기 👀

만약 어떤 예시 코드를 참고하여 작성한 경우라면, 예시 코드를 다시 한번 살펴보고 다른 곳은 없는지 다시 한번 잘 살펴보세요. **사소한 공백, 오타** 조차 문제가 될 수 있습니다. 매우 면밀히 살펴보세요!

<br/>
## 4. 구글링 (Stackoverflow) 🌏

여러분이 개발자도구에서 찾은 오류 메시지나 혹은 (문장 형태보다는) **궁금한 키워드 위주**로 구글에서 검색해보세요. 검색결과에서 `Stackoverflow` 를 최대한 활용해보도록 하세요. 많은 개발자들의 지식인과 같은 서비스입니다.

구글에는 검색 쿼리 기능이 있습니다. 검색어 뒤에 `site:stackoverflow.com` 이라고 입력하면 지정한 웹사이트에 해당하는 검색 결과만 확인할 수 있습니다.

<Callout emoji="📌">
  [구글 검색 쿼리](https://www.itworld.co.kr/news/280537)에 대한 자세한 내용을 찾아보시면 도움이 되는 정보가 많이 있습니다.
</Callout>

<br/>
## 5. 주변 동료에게 도움을 요청하세요. 👯

바보같은 질문은 없습니다. 주변에 있는 동료들과 소통해보세요. 동료들에게 질문을 할 경우, 아래 내용들이 포함되어 있으면 도움이 될 수 있습니다.

1. 본인이 기대했던 결과가 무엇인지에 대한 간략한 설명
2. 실제로 발생한 결과가 무엇인지에 대한 간략한 설명
3. 관련있다고 생각되는 코드 스크린샷
4. 개발자 도구 `console` 스크린샷
